PunchBuddy

Automation app for Pro Tools Punch-In, transport, export and Avid Interplay.
Menu bar app (macOS) with Stream Deck integration, Vocaster control and loudness normalization per EBU R128.

v 1.3  ·  Created by Jens Rühl  ·  06.2026

1 · Settings Menus

All settings are stored under ~/.punchbuddy/settings.json and loaded automatically on the next start. The window opens via Menu bar → Settings. Changes only take effect after clicking Save.

Tab: Tracks (Profile A & B)

PunchBuddy knows two independent track profiles. Profile A is triggered via the main trigger, Profile B via the second trigger (menu or webtrigger /trigger2). This makes it easy to quickly switch between, for example, two different recording setups – such as Speaker 1 vs. Speaker 2.

Stream Deck Section

PunchBuddy ships the Stream Deck plugin and the key launchers bundled directly in the app. Three buttons:

ButtonEffect
Install PluginOpens the bundled com.punchbuddy.control.streamDeckPlugin – Stream Deck shows its own installation dialog. Afterwards the action "PunchBuddy Command" with a command dropdown is available.
Save Plugin (Downloads)Copies the plugin file to ~/Downloads and reveals it in Finder (for sharing/manual installation).
Create Key LaunchersBuilds a tiny .app per command in ~/Applications/PunchBuddy Launchers that sends the command via nc -U to the configured Unix-Socket. Usable in Stream Deck via the built-in "System → Open" action – without installing the plugin.
Why two ways? The plugin (one action, command via dropdown) is convenient but requires Stream Deck ≥ 6.5. The launchers work with any Stream Deck version entirely without a plugin installation. Both hit the same local socket – completely network-free.

Tab: Webtrigger

All triggerable functions run through one central command API (command_dispatch). It is served by two transports:

  • HTTP server (ThreadingMixIn + TCPServer, default port 8899) – for Stream Deck, Companion or any HTTP client.
  • Unix domain socket (/tmp/punchbuddy.sock) – completely without networking (no TCP/loopback), and therefore in principle undetectable by network filters such as Microsoft Defender. Protected via file permissions 0600 (only the logged-in user).

Tab: Presets

There are 8 presets, each of which stores a complete working state. A preset can be loaded manually via the tab or by webtrigger /preset/1 through /preset/8.

Each preset entry stores: Record Tracks A/B, Monitor Tracks A/B, Export Tracks, Play Monitor Tracks, Loudness Tracks as well as all bool/string/int/float settings such as loudness_enabled, interplay_enabled, target_lufs, etc.

Tip: Presets can be loaded via Companion or Stream Deck. This lets you completely switch between different production formats (news broadcast, magazine, …) with a single key press.

2 · Technical Workflows

Startup

1
Set bundle name and process name Before importing rumps/AppKit, CFBundleName = 'PunchBuddy' is written into the InfoDictionary and NSProcessInfo.setProcessName_ is called, so that "Python" does not appear in the Dock and Activity Monitor.
2
Initialize logging & crash diagnostics punchbuddy/log.py creates ~/.punchbuddy/PunchBuddy.log, opens PunchBuddy_crash.log for faulthandler (SIGSEGV/SIGABRT) and installs threading.excepthook / sys.excepthook.
3
Load settings & migration ~/.punchbuddy/settings.json is combined with the default values via a deep merge. If an old ~/.autopunchin/ folder exists, it is migrated once.
4
rumps app loop starts The menu bar app (PunchBuddyApp) is instantiated. The HTTP server, PTSL KeepAlive and Vocaster timer are started as daemon threads/timers.
5
Startup pre-roll check (background thread, 2s delay) Pre-roll is set to OFF (PTSL), the track cache is populated. Both happen in the background so that startup is not blocked.
6
Vocaster initialization USB device detection via libusb. If a Vocaster is found: 48V on (optional) + apply saved routing (optional) + add menu entries.

Punch-In (Record) Record

Triggered via menu, Stream Deck (/trigger or /trigger2) or webtrigger. Runs entirely in a daemon thread so that the UI stays responsive.

1
Guard: already running? state.running_lock prevents parallel recordings. If one is already running, the trigger is ignored.
2
Query track cache (PTSL) Track names from the session cache (loaded once at start). If empty: fresh PTSL call track_list(). Tracks from the settings list are matched against the cache.
3
Pre-roll ON (PTSL, 3 attempts) ensure_preroll_on() reads the current pre-roll status via GetTimelineSelection and sets it with SetTimelineSelection(pre_roll_enabled=TB_True). Fallback to Cmd+K (CGEvent) if PTSL fails.
4
Record enable (PTSL, 3 attempts, 15s timeout each) set_track_record_enable_state(tracks, True). 15s timeout due to possible NEXIS write load. On failure: warning in the log, the workflow continues anyway.
5
Transport arm + Play (PTSL) First it checks whether the transport is already armed (transport_armed()). If not: toggle_record_enable() → toggle_play_state(). Pro Tools starts the recording.
6
Wait until transport is running (max 5s) Polling transport_state() every 50ms until TS_TransportRecording or TS_TransportPlaying.
7
Wait until transport stops (max 10 minutes) Polling every 300ms. On TS_TransportIsStopping (PT writing to NEXIS) it switches to a 3s interval to reduce PTSL load. After 10 consecutive timeouts: engine reset.
8
Transport arm OFF + pre-roll OFF Transport arm is only deactivated if it was activated by PunchBuddy. Pre-roll is reset with restore_preroll().

Play & Play Custom Play

Play (Input Monitor) reads the current transport status. If PT is running → stop path. If PT is stopped → start path: switch the configured (or selected) tracks to input monitor, then start playback. On stop, the monitor tracks are deactivated again.

Play Custom works like Play, but before starting it sets defined mute states on two configurable tracks (typically: mute KH2, keep studio monitoring active). On stop, these mutes are reset again.

Stop Transport

The Stop command is explicitly decoupled from state.running – it also works during a running recording. It uses its own _stop_lock (prevents a double stop, but does not block a parallel Punch-In). toggle_play_state() corresponds to the spacebar in PT.

Move Audio (Track Move) Edit

Moves the entire material of the source tracks onto the target tracks – at an identical time position – entirely via PTSL cut/paste. Triggered via the "Move Audio" menu or webtrigger /move_audio (run_move_audio() in transport.py).

1
Activate edit selection context Via _protools_selection_context: select the Selector tool and enable "Link Timeline and Edit Selection" + "Link Track and Edit Selection". Only then do select_tracks_by_name / set_timeline_selection control the edit selection on which cut/paste operate. The original settings are restored afterwards.
2
Move pairwise per track (src[i] → tgt[i]) The mapping is done explicitly by index, not via PT's top→bottom mapping. Each track is moved individually; before each source track the selection is set to zero length so that empty tracks (stale selection) are skipped correctly.
3
Clip-based cut (PTSL) select_all_clips_on_track(src[i]) creates a clip-accurate edit selection, then cut(). The original clip start is remembered. Settling delays between the steps, since PT has to process every selection change.
4
Paste onto target track at the same position (PTSL) select_tracks_by_name([tgt[i]]) + cursor to the remembered clip start (set_timeline_selection(start, start)), then paste(). Each track lands at its own original position – independent of the start timecode.
Two insights from the live test: (1) PTSL cut/paste operate on the edit selection – without the link options, cut only cuts one track and paste lands in the wrong place. (2) A time-range-based multi-track selection cuts unreliably with material that does not start frame-accurate (subframes, e.g. 10:00:00:11.64). That is why it is clip-based and done per track individually. Verified against Pro Tools: both frame-accurate (A1/A2 → A3/A4) and subframe-offset material (A5/A6 → A7/A8) is moved to the exact position, and the round trip restores the original state.

Full Export Workflow Export

Started via menu or webtrigger. Depending on the configuration it covers: Transport Stop → Consolidate → WAV → Loudness → AAF → Interplay Export. All track and selection control runs via PTSL – no simulated key presses are used for it.

1
Stop transport if active (PTSL) new 06.2026 _ensure_transport_stopped() reads transport_state(). If PT is still running (Record/Play), toggle_play_state() (Stop) is sent, it waits for a confirmed standstill and gives PT 2 s to finalize the file. If the transport is already stopped, the export continues immediately. Protects against session damage if the user triggers the export without a prior stop. Runs in all five export functions.
2
Auto-detect video track (PTSL) _detect_video_track(): looks for a track with type=4 in the PTSL track cache. Fallback to the name entered in the settings.
3
Show tracks + determine video end (PTSL) set_track_hidden_state(tracks, False) shows hidden export tracks. select_all_clips_on_track(video_track) + get_timeline_selection() return the out time (video end).
4
Select tracks & extend selection (PTSL) select_tracks_by_name(export_tracks) marks the export tracks, set_timeline_selection(in_time, out_time) sets the time range, extend_selection_to_target_tracks(export_tracks) extends the edit selection to all target tracks. All via PTSL – no keyboard simulation.
5
Consolidate (PTSL) + wait for completion engine.consolidate_clip() triggers the consolidate via PTSL. The waiting for completion uses AppleScript: _wait_for_consolidate_window_gone() checks the PT window titles (word-start match on "Consolidating"/"Zusammenführen"). The timeout is calculated from the selection length.
6
Wait for WAV file stability _wait_for_consolidated_files(): checks every 100ms whether the WAV files in the Audio Files/ folder are stable (size unchanged for 500ms). Tracks without material do not block.
7
Loudness normalization (optional) For each track in loudness_tracks: read WAV (soundfile), measure LUFS (pyloudnorm), calculate gain correction, apply true-peak limiter, overwrite the file. PT is then asked to reload.
8
AAF export (optional) Via AppleScript UI automation in the PT export dialog. Embedded or Reference depending on the configuration.
9
Interplay export (optional) See its own section below.

Interplay Export Interplay

Entirely via UI automation (AppleScript + CGEvent on the processes Pro Tools and interplayAccess). PTSL offers no API for this.

1
Activate PT, open export dialog AppleScript activates Pro Tools, opens Clip → Export to MediaCentral. PT shows a modal dialog with a progress bar.
2
Monitor PT log via kqueue The PT log (~/Library/Logs/Avid/Pro_Tools_*.txt) is observed live via kqueue (VNODE event). Each new line is checked for SendDialogOnScreen … error msg = …. "successfully" → success; other text → error.
3

5

Interplay Import Interplay

Opens a sequence in Interplay Access and hands it over to Pro Tools. Optionally, the current PT session is closed automatically beforehand (import_close_session). PT confirms the import via a dialog window that is read via AppleScript.

Loudness Normalization (EBU R128) Loudness

Runs after the consolidate, before the AAF export. All steps run in memory – the WAV file is only overwritten at the end.

  1. Find the newest <trackname>*.wav in the Audio Files/ folder (by mtime)
  2. Load audio with soundfile.read()
  3. Measure integrated loudness with pyloudnorm.Meter.integrated_loudness()
  4. Calculate gain difference: gain_db = target_lufs − current_lufs
  5. Apply gain: data *= 10^(gain_db/20)
  6. True-peak limiter: peaks above max_truepeak are scaled back
  7. Overwrite the file with soundfile.write()
  8. Pro Tools: Clip → Refresh via AppleScript so PT reads the new file

Vocaster USB Control Vocaster

PunchBuddy communicates directly via USB (HID) with the Focusrite Vocaster. The vocaster_control.VocasterUSB class encapsulates the USB transactions. All of the logic runs in the main thread via rumps.Timer (every 400ms), because AppKit windows may only be served there.

FunctionFlow
Device detectiondetect_vocaster() searches by vendor/product ID via libusb for a connected Vocaster One or Two.
48V phantom powerA single USB HID report to the register number for the 48V bit. Works instantly, no dialog.
AutogainA USB command starts the Autogain process in the device. An AutogainWindow (native AppKit window) shows progress and result. The drive() tick polls the status via USB every 400ms until Success/Fail/Timeout.
Routingcapture_routing_now() reads the current MUX routing of all channels via USB and saves it as JSON. apply_saved_routing() writes it back.

Threading Model

PunchBuddy is a single-main-thread app (rumps/AppKit) that offloads all long-running operations into daemon threads.

Thread / TimerFileTask

Critical rule: All AppKit/NSStatusItem calls must happen in the main thread. _dispatch_main(fn) in uikit.py uses AppHelper.callAfter() (performSelectorOnMainThread:), which wakes the run loop in a thread-safe way. NSRunLoop.performBlock_() is not thread-safe and must not be used.

PTSL / gRPC Core

PTSL (Pro Tools Scripting Layer) is a gRPC-based API from Avid that listens on port 31416. The Python package ptsl wraps the proto definitions. PunchBuddy manages a singleton instance (_engine_instance) with automatic reconnect.

gRPC deadline wrapper (engine.py:_install_grpc_deadline)

The ptsl package does not pass a timeout to the gRPC stub. PunchBuddy therefore patches raw_client.SendGrpcRequest with a wrapper that sets timeout=_current_grpc_deadline(). The value comes from thread-local storage (_grpc_deadline_tls) and is set by _ptsl_call() before each call.

_ptsl_call() – serialized call

def _ptsl_call(fn, *args, label="", timeout=15.0):
    # Exklusiver Lock: verhindert parallele gRPC-Calls (NEXIS-Delay-Schutz)
    if not _ptsl_lock.acquire(timeout=6.0):
        return False, None

    _grpc_deadline_tls.value = timeout  # Deadline für diesen Call
    try:
        return True, fn(*args)
    except grpc.RpcError as e:
        if e.code() == grpc.StatusCode.DEADLINE_EXCEEDED:
            ...  # Timeout → Engine verwerfen
        _reset_engine()
        return False, e
    finally:
        _ptsl_lock.release()

Track cache

Track names do not change during a session. _get_cached_track_names() loads them once (at start or on the first record trigger) and holds them in memory. The cache is cleared on engine reset / session change via _invalidate_session_tracks(). run_play() still reads tracks live, since the current selection is needed in real time.

AppKit & Main-Thread Dispatch AppKit

# uikit.py – threadsicherer Dispatch auf Main-Thread
def _dispatch_main(fn):
    def _safe():
        try:
            fn()
        except Exception:
            logging.exception("_dispatch_main: Fehler im Main-Thread-Block")
    try:
        from PyObjCTools import AppHelper
        AppHelper.callAfter(_safe)   # = performSelectorOnMainThread:
    except Exception:
        pass

Why not NSRunLoop.performBlock_()? – This method is not thread-safe and does not wake the run loop. Calls from worker threads can be silently lost or lead to race conditions. AppHelper.callAfter() internally uses performSelectorOnMainThread:withObject:waitUntilDone: (wait=False) and is the correct pattern.

CGEvent – Keyboard Control Input

For the few actions without a PTSL API, key presses are sent as CGEvents directly to the Pro Tools PID – this also works when PT is not in the foreground. In practice, only the pre-roll fallback (Cmd+K) remains of this, in case setting it via PTSL SetTimelineSelection fails three times. The selection extension during export does not run via keyboard, but via PTSL extend_selection_to_target_tracks(); the old Shift+Ö helper function (_send_shift_oe) is no longer active in the current code path.

# keys.py
def _send_key_to_app(pid, vk, cmd=False, shift=False):
    src = Quartz.CGEventSourceCreate(kCGEventSourceStateHIDSystemState)
    ev_dn = Quartz.CGEventCreateKeyboardEvent(src, vk, True)
    if cmd:
        Quartz.CGEventSetFlags(ev_dn, kCGEventFlagMaskCommand)
    Quartz.CGEventPostToPid(pid, ev_dn)
    Quartz.CGEventPostToPid(pid, Quartz.CGEventCreateKeyboardEvent(src, vk, False))

VK constants (macOS hardware codes, layout-independent): _VK_K=40 (Cmd+K = pre-roll fallback), _VK_F2=120 (Interplay Rename), _VK_ESC=53, _VK_RETURN=36. CGEvents are also used in the Interplay UI automation (F2/Tab/Cmd+A/C/V), since Interplay Access has no PTSL API.

AppleScript / UI Automation UI

AppleScript via osascript (subprocess) is used for actions that have no PTSL API:

  • Read PT window titles (name of every window) – e.g. to wait for the end of the consolidate operation
  • Trigger the PT export menu (Clip → Export to MediaCentral). Consolidate itself runs via PTSL consolidate_clip(), not via AppleScript.
  • Control Interplay Access (Tab navigation, F2 rename, Cmd+C/V)
  • Read static text from PT dialog windows (export result)
  • Activate an app (tell application "interplayAccess" to activate)

All AppleScript calls are wrapped by _run_applescript(script, timeout=30), which starts a subprocess.run() process. On timeouts, the function returns an error return code.

Accessibility permission: AppleScript access to System Events requires that PunchBuddy (or Python) is enabled under System Settings → Privacy → Accessibility.

kqueue – File Monitoring macOS

kqueue is the macOS kernel event notification mechanism. PunchBuddy uses it to monitor the PT log file without polling:

# export.py – Vereinfacht
fd = os.open(log_path, os.O_RDONLY | os.O_NONBLOCK)
kq = select.kqueue()
ke = select.kevent(fd,
    filter=select.KQ_FILTER_VNODE,
    flags=select.KQ_EV_ADD | select.KQ_EV_ENABLE | select.KQ_EV_CLEAR,
    fflags=select.KQ_NOTE_WRITE | select.KQ_NOTE_EXTEND)

while time.time() < deadline:
    events = kq.control([ke], 4, min(remaining, 2.0))  # blockiert bis Event
    if events:
        # Neue Bytes lesen, auf Marker prüfen
        ...

Advantages over polling: instant reaction to a file event (no fixed 0.3s interval), no CPU load between write operations. Fallback to 0.3s polling if kqueue is unavailable for any reason.

Control API: Dispatcher + Transports API

At the core is one command table command_dispatch(command, args) → (ok, msg) (_command_table()). ONE entry per function (record_a/b, play, play_custom, goto_start, move_audio, import, all exports, preset, vocaster_*, ping/status/list). All transports call this dispatcher exclusively – so HTTP and socket automatically share debounce and export serialization.

Transport 1 – HTTP: embedded ThreadingMixIn + TCPServer. An immediate 200 OK per trigger request, the function continues queued. Socket timeout (30 s) against slow clients. On a bind to 0.0.0.0 (LAN), a token (secrets.token_urlsafe(16)) is generated automatically; requests without the correct token receive 401 (comparison is timing-safe).

Transport 2 – Unix domain socket: AF_UNIX/SOCK_STREAM on /tmp/punchbuddy.sock, file permissions 0600. No IP stack → invisible to network filters (Defender). Line protocol, threaded accept loop, clean shutdown via atexit + quit/signal handler + stale socket cleanup.

def command_dispatch(self, command, args=None):
    cmd = _COMMAND_ALIASES.get(command.lower(), command.lower())
    handler = self._command_table().get(cmd)
    if handler is None: return (False, f"unknown command: {command}")
    return handler(args)        # → (ok, msg)

Stream Deck Integration Stream Deck

Two network-free ways, both against the Unix-Socket:

Settings System Config

Settings are stored as JSON under ~/.punchbuddy/settings.json. On load, a recursive deep merge with the default values is performed: newly added keys from the code are added automatically, existing user values are preserved.

def _deep_merge(default, override):
    """Neue Keys aus default werden ergänzt; override gewinnt bei Konflikten."""
    if isinstance(default, dict) and isinstance(override, dict):
        out = dict(default)
        for k, v in override.items():
            out[k] = _deep_merge(default[k], v) if k in default else v
        return out
    return override

Preset entries additionally get a template merge (_PRESET_TEMPLATE) so that newly added preset keys also appear in stored old presets.

Crash Diagnostics Debug

Three layers catch errors that would otherwise disappear without a trace:

MechanismFileWhat it catches
faulthandlerlog.pyNative crashes (SIGSEGV, SIGABRT) – writes C-level stack traces of all Python threads into PunchBuddy_crash.log. Indispensable for AppKit/ObjC errors.
threading.excepthooklog.pyUnhandled exceptions in daemon threads. Without the hook, threads die silently.
sys.excepthooklog.pyUnhandled exceptions in the main thread (before the rumps run loop ends).

The diagnostics script collect_diagnostics.py (or PunchBuddy_Diagnose.app from the DMG) collects all relevant logs, PT crash reports, system info and the faulthandler output into a single text file on the desktop.

Important for the build: The Universal2 build (build_dmg.sh) requires two Python installations – one native ARM64 and one x86_64 – each with its own packages (numpy, scipy etc.). The Intel build (build_dmg_intel.sh) runs with the framework Python 3.14 (x86_64) and bundles libusb automatically. For the studio machine (Intel Mac), the Intel build is the correct one.
